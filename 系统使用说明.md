# 单细胞 ANN 检索系统使用说明

本文档说明当前中期版本已经实现的功能、运行方式、页面操作流程、API 调用方式，以及可用于测试的 `cell_id`。

## 1. 当前已实现功能

当前系统是前后端分离的 Web 应用。

已实现功能：

- 读取本地单细胞数据文件：`data/liver.h5ad`
- 使用 `obsm["X_pca"]` 作为 ANN 检索向量
- 使用 `obsm["X_umap"]` 作为二维可视化坐标
- 读取细胞元信息：`cell_type`、`disease`、`AgeGroup`、`tissue`
- 使用 FAISS 构建 IVF_FLAT 近似最近邻索引
- 优先使用 GPU FAISS，无法使用 GPU 时可降级到 CPU FAISS
- 输入 `cell_id` 查询 Top-K 相似细胞
- 返回排名、细胞 ID、距离、相似度、细胞类型、疾病、年龄组、组织、UMAP 坐标
- 前端展示数据集状态、索引状态、查询结果表格和 UMAP 散点图
- 查询日志写入：`logs/query_log.jsonl`

暂未实现功能：

- 用户登录和权限管理
- 条件检索
- 多数据集管理
- 完整性能评测报告
- 结果导出

## 2. 启动系统

### 2.1 启动后端

在项目根目录执行：

```powershell
conda activate single-cell-ann
python -m backend.app
```

后端默认地址：

```text
http://127.0.0.1:5000
```

如果看到以下内容，说明后端启动成功：

```text
Running on http://127.0.0.1:5000
```

### 2.2 启动前端

另开一个终端，在项目根目录执行：

```powershell
cd frontend
npm install
npm run dev
```

前端默认地址通常是：

```text
http://127.0.0.1:5173
```

如果 `5173` 被占用，Vite 可能自动切换到 `5174`。当前后端已经允许 `5173-5176`，所以可以直接打开 Vite 输出的地址。

## 3. 页面使用流程

打开前端页面后，按下面顺序操作：

1. 点击“加载数据”
   - 后端会读取 `data/liver.h5ad`
   - 页面会显示细胞数量、向量维度和示例细胞 ID

2. 点击“构建索引”
   - 后端会基于 PCA 向量构建 FAISS IVF_FLAT 索引
   - 页面会显示索引状态、索引类型、GPU/CPU 模式和 `nprobe`

3. 输入 `cell_id`
   - 可以使用页面显示的示例细胞 ID
   - 也可以使用本文档第 6 节中的示例 `cell_id`

4. 设置 Top-K
   - 默认是 `10`
   - 建议演示时使用 `5` 或 `10`

5. 点击“查询”
   - 页面会展示 Top-K 相似细胞表格
   - 页面会展示查询耗时和结果数量
   - UMAP 图中会高亮查询细胞和 Top-K 相似细胞

## 4. API 使用方式

### 4.1 健康检查

```powershell
curl http://127.0.0.1:5000/api/health
```

可查看：

- 后端是否正常
- 数据文件是否存在
- FAISS 是否可用
- 当前 FAISS 是 GPU 还是 CPU 模式

### 4.2 加载数据

```powershell
curl -X POST http://127.0.0.1:5000/api/datasets/load
```

成功后会返回：

- `cell_count`
- `vector_dim`
- `metadata_fields`
- `sample_cell_ids`

当前数据集应返回：

```text
cell_count = 69032
vector_dim = 30
```

### 4.3 查看当前数据集状态

```powershell
curl http://127.0.0.1:5000/api/datasets/current
```

### 4.4 构建索引

```powershell
curl -X POST http://127.0.0.1:5000/api/index/build
```

默认参数：

```text
index_type = IVF_FLAT
metric = l2
nlist = 256
nprobe = 16
```

### 4.5 查看索引状态

```powershell
curl http://127.0.0.1:5000/api/index/status
```

### 4.6 查询相似细胞

示例：

```powershell
curl -X POST http://127.0.0.1:5000/api/search `
  -H "Content-Type: application/json" `
  -d "{\"cell_id\":\"AAACCTGAGCAGGTCA-1_2\",\"top_k\":5}"
```

返回内容包括：

- 查询细胞
- 查询耗时
- 索引状态
- Top-K 结果
- 每个命中细胞的元信息和 UMAP 坐标

### 4.7 获取 UMAP 抽样点

```powershell
curl http://127.0.0.1:5000/api/visualization/cells?limit=5000
```

该接口用于前端绘制 UMAP 散点图。

## 7. 如何查看更多 cell_id

如果需要查看更多 `cell_id`，可以运行：

```powershell
conda activate single-cell-ann
python -c "import h5py; f=h5py.File('data/liver.h5ad','r'); print([x.decode() for x in f['obs']['_index'][:100]])"
```

如果要按细胞类型统计：

```powershell
conda activate single-cell-ann
python -c "import h5py, collections; f=h5py.File('data/liver.h5ad','r'); cats=[x.decode() for x in f['obs']['cell_type']['categories'][:]]; codes=f['obs']['cell_type']['codes'][:]; print(collections.Counter(cats[int(c)] for c in codes if int(c)>=0))"
```

## 8. 常见问题

### 8.1 页面显示 Network Error

常见原因：

- 后端没有启动
- 后端不是运行在 `http://127.0.0.1:5000`
- 前端端口被 Vite 切换了，但后端没有重启加载新的 CORS 配置

处理方式：

```powershell
conda activate single-cell-ann
python -m backend.app
```

然后刷新前端页面。

### 8.2 构建索引按钮不可用

需要先点击“加载数据”，数据加载成功后才能构建索引。

### 8.3 查询按钮不可用

需要满足三个条件：

- 已加载数据
- 已构建索引
- 已输入合法 `cell_id`

### 8.4 FAISS 不可用

先检查环境：

```powershell
conda activate single-cell-ann
python -c "import faiss; print(faiss.__version__); print(faiss.get_num_gpus())"
```

如果不能导入 `faiss`，需要重新创建 Conda 环境：

```powershell
conda env create -f environment.yml
```
