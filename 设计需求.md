# 单细胞 ANN 检索系统设计需求

本文档结合课程团队作业要求、《第二次作业：需求分析》和《第三次作业：软件设计》，用于明确本项目中期和结项阶段代码需要完成的事情。

## 1. 项目定位

项目目标是开发一个面向单细胞高维向量数据的近似最近邻检索系统。系统需要把单细胞样本表示为统一维度的向量，并通过精确检索和 ANN 检索返回 Top-K 相似细胞。

系统最终形态为 Web 应用，用户可以通过页面完成数据集选择、查询细胞输入、查询向量上传、检索参数设置、相似细胞结果查看、查询耗时查看、索引状态查看、可视化分析和性能评测。

## 2. 设计依据

第二次作业给出了需求分析层面的设计，重点包括：

- 五类用户角色：在线访客、普通注册用户、专业研究人员、数据维护者、系统管理员。
- 七个功能模块：用户与权限管理、数据接入与维护、数据预处理、索引管理、查询检索、结果展示与分析、系统监控与日志。
- 六类核心数据表：User、Dataset_Info、Cell_Vector、Index_Info、Query_Log、Benchmark_Result。
- 非功能要求：准确性、及时性、可扩充性、易用性、易维护性、标准性、先进性和安全性。

第三次作业给出了软件设计层面的设计，重点包括：

- 四层系统结构：UserLayer、ViewLayer、BizLayer、DataLayer。
- 八个代码包：user、dataset、preprocessing、index、search、evaluation、visualization、system。
- 关键类与服务：User、Dataset、CellSample、PreprocessingPipeline、VectorIndex、AnnIndex、ExactIndex、IndexManager、QueryRequest、SearchResult、EvaluationTask、VisualizationService、Repository 等。
- 关键流程：注册登录、数据导入校验、数据预处理、索引构建、相似细胞查询、结果可视化、性能评测、管理员用户管理。
- 数据集生命周期：上传、校验、预处理、索引构建、上线使用、下线删除。

## 3. 用户角色与权限边界

### 3.1 在线访客

在线访客未登录系统，主要用于浏览系统介绍、功能概览、示例结果和登录注册入口。

中期可暂不实现完整访客体系，但结项时建议保留公开首页或示例页面。

### 3.2 普通注册用户

普通注册用户是基础业务用户，主要完成常规 Top-K 查询。

需要支持：

- 登录后进入检索页面。
- 选择数据集。
- 输入查询细胞 ID 或查询向量。
- 设置基础参数，如 `top_k`、检索方式、距离度量。
- 查看相似细胞列表、距离或相似度、查询耗时和基础可视化结果。

### 3.3 专业研究人员

专业研究人员是高级业务用户，重点关注检索质量、检索效率和结果对比。

需要支持：

- 高级参数设置。
- 精确检索与 ANN 检索切换。
- 批量查询。
- ANN 与精确检索结果对比。
- Recall、QPS、平均延迟、P50、P99、内存占用等评测指标查看。
- 评测报告或检索结果导出。

### 3.4 数据维护者

数据维护者负责让数据从原始文件变成可检索资源。

需要支持：

- 上传或登记数据集文件。
- 校验数据格式、字段完整性、维度一致性和缺失情况。
- 维护数据集来源、物种、组织、细胞类型、标签等元信息。
- 触发预处理流程。
- 查看数据集处理状态。

### 3.5 系统管理员

系统管理员负责平台治理和后台维护。

需要支持：

- 用户账号管理。
- 用户角色分配。
- 权限控制。
- 索引构建、保存、加载、切换、删除。
- 查看索引状态、服务状态、运行日志、异常日志和性能记录。

## 4. 总体架构

系统采用分层设计：

| 层次 | 代码职责 |
| --- | --- |
| UserLayer | 对应在线访客、普通注册用户、专业研究人员、数据维护者和系统管理员 |
| ViewLayer | Web 页面、表单、结果表格、可视化图表和管理后台 |
| BizLayer | 用户认证、数据管理、预处理、索引管理、查询检索、性能评测、结果展示、日志监控 |
| DataLayer | 用户库、数据集元信息库、细胞向量文件、索引文件、查询日志、评测结果 |

部署上可先采用单机 Web 应用形式，逻辑上区分：

- 客户端访问节点：浏览器页面。
- Web 服务节点：HTTP 路由、静态资源、接口入口。
- 核心应用服务节点：认证、数据、预处理、索引、检索、评测等业务逻辑。
- 数据与索引资源节点：数据库、数据文件、向量文件、索引文件、日志文件。
- 监控与日志支撑环境：查询日志、操作日志、异常日志和评测记录。

中期可以单进程部署；结项时至少要在代码结构上体现前端入口、业务服务和数据存储的分离。

## 5. 推荐代码包与职责

### 5.1 `user` 用户与权限包

核心对象：

- `User`：保存 `user_id`、`username`、`password_hash`、`email`、`role`、`status`。
- `NormalUser`：普通查询用户。
- `Researcher`：支持高级检索和评测。
- `Admin`：负责用户和权限管理。
- `AuthController` 或 `AuthService`：处理注册、登录、身份认证、权限校验。
- `UserRepository`：负责用户数据存取。

中期要求：

- 可简化为基础登录或管理员默认账号。
- 若时间紧张，中期可以先不做完整角色体系，但接口和代码目录应预留。

结项要求：

- 实现注册、登录、密码加密存储、角色划分和权限控制。
- 区分普通用户、研究人员、数据维护者、管理员的功能入口。
- 支持管理员查看、修改、禁用或启用用户。

### 5.2 `dataset` 数据管理包

核心对象：

- `Dataset`：表示数据集，包含 `dataset_id`、名称、描述、细胞数量、基因数量、上传时间、状态。
- `CellSample`：表示单个细胞样本，包含 `cell_id`、`dataset_id`、向量和元数据。
- `Metadata`：保存来源、物种、组织、细胞类型等信息。
- `Vector`：保存维度、数值、是否归一化等信息。
- `DatasetController` 或 `DatasetService`：处理导入、校验、删除、列表查询。
- `DatasetRepository`：保存和读取数据集记录。

中期要求：

- 支持读取给定 `.h5ad` 数据文件。
- 能从 `X` 或 `obsm["X_pca"]` 等字段中提取细胞向量。
- 保存细胞 ID 与向量下标的映射。
- 读取细胞元信息，如细胞类型、组织来源、标签等。

结项要求：

- 支持数据集新增、删除、下线、切换和状态查看。
- 支持数据集元信息维护。
- 支持多数据集管理，每个数据集对应自己的向量文件和索引文件。

### 5.3 `preprocessing` 数据预处理包

核心对象：

- `PreprocessingPipeline`：串联多个预处理步骤。
- `ProcessingTask`：记录一次预处理任务的状态、开始时间、结束时间和失败原因。
- `PreprocessingStep`：预处理步骤统一接口。
- `FormatValidationStep`：校验文件结构、字段和维度。
- `DataCleaningStep`：处理缺失值、异常值和重复记录。
- `NormalizationStep`：完成标准化或归一化。
- `FeatureSelectionStep`：完成特征筛选、PCA 或其他 embedding 提取。
- `TaskStatus`：`CREATED`、`RUNNING`、`SUCCESS`、`FAILED`。

中期要求：

- 至少完成数据读取、基础格式校验和向量提取。
- 如果老师提供的数据已经预处理完成，中期可以直接使用已有 PCA 向量，但需要在代码中说明向量来源。

结项要求：

- 实现可追踪的预处理任务。
- 能展示或记录预处理状态。
- 支持格式校验、清洗、归一化、特征处理、向量生成和向量保存。
- 任一步骤失败时记录失败原因并向前端返回清晰提示。

### 5.4 `index` 索引管理包

核心对象：

- `VectorIndex`：索引抽象基类，定义 `build(dataset)`、`search(query, top_k)`、`save(path)`、`load(path)`、`get_status()`。
- `AnnIndex`：ANN 索引抽象类，定义 `approximate_search(query, top_k)`、`tune_parameter(params)`。
- `ExactIndex`：精确检索基线，用暴力扫描实现 KNN。
- `HnswIndex`：HNSW 近似索引，参数包括 `m`、`ef_construction`、`ef_search`。
- `IvfIndex`：IVF 近似索引，参数包括 `cluster_count`、`n_probe`。
- `IndexParameter`：保存索引参数。
- `IndexManager`：负责索引创建、构建、保存、加载、切换和状态维护。
- `IndexRepository`：保存索引记录和文件路径。

中期要求：

- 至少实现一种 ANN 索引或接入一种检索库。
- 建议优先使用 `hnswlib` 或 `FAISS`。
- 支持构建索引、执行 Top-K 查询、返回索引状态。
- 支持一种距离度量，如欧氏距离或余弦距离。

结项要求：

- 支持精确索引和 ANN 索引。
- 支持至少一种可配置 ANN 索引，推荐 HNSW；如时间允许可增加 IVF。
- 支持索引保存、加载、切换、删除和重建。
- 支持为不同数据集维护不同索引。
- 记录索引构建时间、参数、状态和文件路径。

### 5.5 `search` 查询检索包

核心对象：

- `QueryRequest`：封装 `request_id`、`user_id`、`dataset_id`、`query_cell_id`、`query_vector`、`top_k`、`index_type`、`metric`、过滤条件。
- `SearchHit`：表示单条命中结果，包含 `cell_id`、`rank`、`distance`、`similarity` 和元信息。
- `SearchResult`：保存结果集合、查询耗时、索引状态和创建时间。
- `SearchService`：负责校验请求、选择索引、执行检索、封装结果、记录查询日志。
- `ResultRepository`：缓存或持久化查询结果。

中期要求：

- 支持输入查询细胞 ID。
- 支持设置 `top_k`。
- 返回 Top-K 相似细胞 ID、距离或相似度、细胞元信息。
- 返回查询耗时。
- 对细胞 ID 不存在、索引未构建、参数不合法等情况给出错误提示。

结项要求：

- 支持查询细胞 ID 和查询向量两类输入。
- 支持条件检索，例如限定细胞类型、组织来源、数据集来源等。
- 支持精确检索、ANN 检索和批量查询。
- 支持结果导出。
- 每次查询都写入 `Query_Log`。

### 5.6 `evaluation` 性能评测包

核心对象：

- `EvaluationTask`：表示一次评测任务，包含数据集、ANN 索引、精确索引、查询数量和状态。
- `EvaluationMetric`：保存 Recall、QPS、平均延迟、P50、P99、内存占用等指标。
- `EvaluationReport`：生成和导出评测报告。
- `EvaluationService`：负责批量执行精确检索与 ANN 检索，并计算指标。

中期要求：

- 至少记录单次查询耗时。
- 若时间允许，提供 ANN 和精确检索的一次简单耗时对比。

结项要求：

- 支持批量评测。
- 计算 Recall@K、QPS、平均延迟、P50、P99、内存占用、索引构建时间。
- 对比 ANN 检索与精确检索。
- 支持评测结果展示和导出。
- 将评测结果写入 `Benchmark_Result`。

### 5.7 `visualization` 可视化展示包

核心对象：

- `ResultView`：负责结果表格展示。
- `ChartView`：负责散点图、指标图、投影图展示。
- `VisualizationService`：生成结果表格、向量二维投影、性能指标图。

中期要求：

- 至少展示 Top-K 结果表格。
- 表格字段包括排名、细胞 ID、距离或相似度、细胞类型或标签、所属数据集。

结项要求：

- 至少提供一种可视化结果，推荐二维 PCA、UMAP 或 t-SNE 散点图。
- 高亮查询细胞和 Top-K 相似细胞。
- 支持按细胞类型、组织来源等元信息上色。
- 鼓励支持点击图上细胞直接发起查询。
- 展示性能评测图表。

### 5.8 `system` 系统支撑包

核心对象：

- `DataFile`：统一表示数据文件、元数据文件、索引文件。
- `QueryLog`：记录查询行为。
- `SystemLog`：记录系统操作和异常。
- `Repository`：定义 `save(obj)`、`find_by_id(id)`、`delete(id)`。
- `DatasetRepository`、`IndexRepository`、`ResultRepository`、`UserRepository`：分别管理不同数据对象。

中期要求：

- 至少记录查询日志或在后端控制台输出查询参数、耗时和结果数量。
- 重要错误需要能被定位。

结项要求：

- 实现查询日志、操作日志和异常日志。
- 支持管理员查看索引状态、服务状态和历史性能。
- 日志中避免保存密码、敏感路径等敏感信息。

## 7. 数据集生命周期

代码中应明确维护数据集状态，建议状态流转如下：

```text
已上传
  -> 待校验
  -> 校验失败 / 待预处理
  -> 预处理中
       -> 数据读取
       -> 数据清洗
       -> 数据归一化
       -> 特征处理
       -> 向量生成
       -> 向量保存
  -> 预处理失败 / 已预处理
  -> 待构建索引
  -> 索引构建中
  -> 索引构建失败 / 索引构建完成
  -> 已可用
  -> 已下线
  -> 已删除
```

中期可以只实现简化状态：`RAW`、`LOADED`、`INDEXED`、`ERROR`。

结项应实现较完整状态，用于数据集管理页面、索引管理页面和错误提示。

## 8. 后端接口建议

### 8.1 用户接口

- `POST /api/auth/register`：用户注册。
- `POST /api/auth/login`：用户登录。
- `POST /api/auth/logout`：退出登录。
- `GET /api/users`：管理员查看用户列表。
- `PATCH /api/users/{user_id}/role`：管理员修改用户角色。
- `PATCH /api/users/{user_id}/status`：管理员启用或禁用用户。

### 8.2 数据集接口

- `GET /api/datasets`：查看数据集列表。
- `POST /api/datasets`：新增或上传数据集。
- `GET /api/datasets/{dataset_id}`：查看数据集详情。
- `DELETE /api/datasets/{dataset_id}`：删除数据集。
- `POST /api/datasets/{dataset_id}/activate`：切换当前数据集。
- `PATCH /api/datasets/{dataset_id}/metadata`：修改数据集元信息。

### 8.3 预处理接口

- `POST /api/datasets/{dataset_id}/validate`：执行数据校验。
- `POST /api/datasets/{dataset_id}/preprocess`：触发预处理。
- `GET /api/preprocess/tasks/{task_id}`：查看预处理任务状态。

### 8.4 索引接口

- `POST /api/index/build`：构建索引。
- `POST /api/index/load`：加载索引。
- `POST /api/index/switch`：切换索引。
- `DELETE /api/index/{index_id}`：删除索引。
- `GET /api/index/status`：查看当前索引状态。
- `GET /api/datasets/{dataset_id}/indexes`：查看某数据集的索引列表。

### 8.5 检索接口

- `POST /api/search`：普通 Top-K 检索。
- `POST /api/search/conditional`：带条件过滤的 Top-K 检索。
- `POST /api/search/batch`：批量检索。
- `GET /api/search/results/{result_id}`：查看历史检索结果。
- `GET /api/search/results/{result_id}/export`：导出检索结果。

### 8.6 评测接口

- `POST /api/evaluate`：创建评测任务。
- `GET /api/evaluate/tasks/{task_id}`：查看评测状态。
- `GET /api/evaluate/reports/{report_id}`：查看评测报告。
- `GET /api/evaluate/reports/{report_id}/export`：导出评测报告。

### 8.7 可视化接口

- `GET /api/visualization/cells`：获取细胞二维坐标和元信息。
- `GET /api/visualization/search-results/{result_id}`：获取查询结果可视化数据。
- `POST /api/visualization/cell-query`：从图上选择细胞并发起查询。

### 8.8 日志与监控接口

- `GET /api/logs/query`：查看查询日志。
- `GET /api/logs/system`：查看系统操作日志。
- `GET /api/logs/errors`：查看异常日志。
- `GET /api/system/status`：查看服务状态、索引状态和资源状态。

## 9. 中期代码需求

中期目标是完成一条可运行的核心检索链路，证明系统已经具备“从单细胞数据到 ANN Top-K 检索结果”的基本能力。

### 9.1 中期必须完成

1. 项目基础结构
   - 建立后端项目目录。
   - 提供依赖文件，如 `requirements.txt`。
   - 提供启动说明。
   - 使用 Git 管理代码，并提交到 GitHub。

2. 数据读取
   - 支持读取 `.h5ad` 单细胞数据。
   - 能读取表达矩阵或已有低维向量，如 PCA embedding。
   - 能读取细胞 ID 和基础元信息。
   - 对数据维度、空数据、字段缺失进行基础校验。

3. 向量化表示
   - 为每个细胞生成或提取统一维度向量。
   - 建立 `cell_id -> vector_index` 的映射。
   - 建立 `vector_index -> cell_metadata` 的映射。
   - 明确记录 embedding 来源，如 `X`、`obsm["X_pca"]` 或其他字段。

4. ANN 索引构建
   - 至少实现一种 ANN 算法或检索库。
   - 推荐 HNSW 或 FAISS。
   - 能基于细胞向量构建索引。
   - 能查看索引是否已构建、向量数量、维度和索引类型。

5. Top-K 相似细胞检索
   - 支持输入查询细胞 ID。
   - 支持设置 `top_k`。
   - 返回 Top-K 相似细胞。
   - 结果包含细胞 ID、排名、距离或相似度、细胞标签或类型。
   - 返回查询耗时。

6. Web 页面或接口展示
   - 能通过 Web 页面或 API 完成查询。
   - 页面至少包含查询输入、参数设置、查询按钮和结果表格。
   - 能展示查询耗时和索引状态。

7. 中期展示材料
   - 每组展示 5-8 分钟。
   - 展示已完成功能。
   - 展示界面演示，现场或录制视频均可。

### 9.2 中期建议完成

- 简单数据集列表。
- 简单查询日志记录。
- 精确检索基线，用于和 ANN 查询耗时对比。
- 简单二维散点图，用于展示查询细胞和 Top-K 结果。

### 9.3 中期验收路径

中期演示时建议按以下路径操作：

## 11. 非功能需求

### 11.1 准确性

- ANN 检索结果应尽量接近精确检索结果。
- 精确检索作为 ANN 评测基线。
- 检索结果中的细胞 ID、标签、距离、所属数据集必须正确对应。
- 数据预处理应保证向量维度一致，避免映射错误。

### 11.2 及时性

- 常规查询应能较快返回 Top-K 结果。
- ANN 平均响应时间应明显优于精确检索。
- 索引构建和加载应在合理时间内完成。
- 上传、预处理、建索引等耗时任务需要返回状态。

### 11.3 可扩充性

- 索引层应支持新增 ANN 算法。
- 数据层应支持扩展更多格式和来源。
- 权限层应支持新增角色。
- 模块之间通过服务接口和 Repository 隔离。

### 11.4 易用性

- 查询入口清晰。
- 参数有合理默认值。
- 错误提示清楚。
- 普通查询流程步骤少。
- 高级参数对研究人员可见，但不干扰普通用户。

### 11.5 易维护性

- 代码目录与功能模块一致。
- 配置集中管理。
- 关键对象具有统一编号和状态。
- 关键流程有日志。
- 预处理、索引构建、查询执行具备异常处理。

### 11.6 标准性

- 接口命名统一。
- 数据表和字段命名统一。
- 用户角色、索引状态、数据集状态使用枚举。
- 页面术语保持一致。

### 11.7 安全性

- 密码不能明文保存。
- 不同角色不能越权访问。
- 删除数据集、删除索引、修改权限等操作需要确认。
- 上传文件和用户输入需要校验。
- 日志中避免暴露密码和敏感路径。

## 14. 加分方向

在基础要求完成后，可以考虑：

- 多数据集联合检索：支持多个单细胞数据集联合建索引，实现跨数据集搜索。
- ANN 算法改进：改进现有 ANN 算法，并在召回率、查询时间或内存占用方面给出对比。
- RAG + 单细胞数据库：支持自然语言查询细胞，并由 AI 辅助解释检索结果。
- 检索历史与结果复现：保存查询参数、索引版本和结果快照。
- GPU 加速：使用 FAISS GPU 版本提升大规模检索性能。
- 任务队列：将预处理、索引构建和评测任务异步化。
